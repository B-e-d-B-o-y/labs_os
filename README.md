# OS Labs: Система мониторинга температуры и сбора данных

Кроссплатформенный проект на C/C++, разработанный в рамках курса по операционным системам. Система считывает данные о температуре через последовательный порт (Serial Port), валидирует их, сохраняет в базу данных SQLite с расчетом средних значений и предоставляет веб-интерфейс для просмотра статистики.

## Возможности

- **Кроссплатформенность:** Полная поддержка Windows (WinAPI) и POSIX-систем (Linux, macOS через pthreads/termios).
- **Работа с железом:** Чтение данных из последовательного порта (9600 бод, 8 бит, без контроля четности, 1 стоп-бит).
- **Валидация данных:** Проверка формата температуры ([-]ddd.d) и верификация по контрольной сумме (checksum).
- **Многопоточность и синхронизация:** Использование кроссплатформенных потоков и семафоров для безопасной одновременной работы с ресурсами.
- **Хранение данных:** Интеграция с SQLite3 для сохранения сырых измерений, а также расчет средних значений за час и день.
- **Веб-сервер:** Встроенный HTTP-сервер для отдачи статического контента и данных.

## Структура проекта

Проект разделен на логические этапы (лабораторные работы):

- **Lab1/** – Базовая настройка окружения и тестовая программа.
- **Lab2/** – Основы сборки и модульности. Создание библиотеки, приложения и скриптов автоматизации сборки и Git.
- **Lab3/** – Многопоточность и IPC. Кроссплатформенная реализация потоков и семафоров, логирование с указанием PID и временных меток.
- **Lab4/** – Работа с периферией. Реализация кроссплатформенного класса SerialPort и утилит для парсинга и валидации данных.
- **Lab5/** – Финальная интеграция. Основной сервер, объединяющий чтение из порта, обработку, БД и веб-интерфейс.

## Требования

- **CMake** версии 3.16 или выше.
- **Компилятор C++17** (GCC, Clang или MSVC).
- **Git** (для использования скриптов инициализации).

## Сборка и запуск

Проект использует CMake. Вы можете использовать готовые скрипты или выполнить сборку вручную.

### Вариант 1: Использование скрипта (Рекомендуется)

Для Linux/macOS:
```bash
chmod +x git.sh
./git.sh

```

Для Windows (Command Prompt):

```cmd
git_run.cmd

```

### Вариант 2: Ручная сборка через CMake

```bash
# 1. Создайте директорию для сборки
mkdir build
cd build

# 2. Сконфигурируйте проект
cmake ..

# 3. Скомпилируйте проект
cmake --build .

```

## Использование

1. Убедитесь, что у вас есть доступ к последовательному порту, к которому подключен датчик.
2. Запустите исполняемый файл. Программа автоматически инициализирует БД, откроет порт и запустит сервер.
3. Откройте браузер и перейдите по адресу `http://localhost:[порт]` для доступа к интерфейсу.

## Технологический стек

* **Языки:** C, C++17
* **Система сборки:** CMake
* **База данных:** SQLite3
* **Сетевое взаимодействие:** Сокеты (POSIX / Winsock)
* **Синхронизация:** POSIX Semaphores / Windows Semaphores
* **Потоки:** pthread / CreateThread

## Формат данных

Ожидаемый формат входящей строки: `[-]ddd.d;checksum`

* `[-]ddd.d` – температура (например, 23.4 или -5.0).
* `checksum` – сумма ASCII-кодов символов строки температуры.
